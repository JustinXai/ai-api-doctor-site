为什么需要这个工具？

很多 AI API 中转站采用预扣费机制。请求失败、空回复、上游 503、超时、缓存命中等情况，都可能让用户难以判断实际扣费是否合理。AI API Doctor 通过可复现信号帮助用户和站长沟通。

五个检测维度

扣费完整性
检测失败请求和空回复是否最终扣费，等待 10 秒确认预扣返还。
模型联通
验证 Base URL、API Key、模型权限和 HTTP 状态是否正常。
usage 完整性
检查响应是否返回 prompt_tokens、completion_tokens 和 total_tokens。
缓存命中
发送两次相同请求，检测 cached_tokens / cache_read_input_tokens 是否返回。
价格核对
根据 usage 和价格计算理论成本，与实际扣费对比，判断是否存在异常。

为什么做 AI API Doctor？

我自己在做 AI API 服务时发现，很多中转站采用预扣费机制：请求开始时先扣额度，结束后再根据 usage 结算返还。

这套机制本身没问题，但如果失败请求、空回复、上游 503、无效模型、缓存命中等场景处理不好，用户可能会遇到"没有拿到有效回答，但额度发生变化"的情况。

AI API Doctor 的目标不是给任何站点定罪，而是把扣费、usage、缓存和模型表现变成一张可复现的体检报告，让用户和站长可以基于证据沟通。

AI API Doctor 不推荐、不排名任何中转站。报告只展示本次测试中的可复现信号，不证明服务商主观故意。
