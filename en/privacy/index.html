Privacy Policy

Learn how AI API Doctor handles your data.

1. What Data Does AI API Doctor Process?

AI API Doctor is a local diagnostic tool. Data processed by the extension may include:

  • Base URL entered by the user
  • API Key entered by the user
  • Model ID
  • Raw quota (from New API / One API)
  • Balance change records
  • HTTP status codes
  • Diagnostic report content

2. How API Keys Are Stored

API Keys in the Chrome extension are stored locally in your browser via chrome.storage.local, used only to send diagnostic requests to the Base URL you provided.

The web report generator does not require you to enter an API Key. It only accepts public information such as raw quota, HTTP status, and model name.

3. Scope of API Key Usage

API Keys are used only to send user-initiated diagnostic requests to the Base URL provided by the user.

AI API Doctor does not upload API Keys, raw quota, or diagnostic reports to AI API Doctor servers.

4. Automatic Report Desensitization

When the extension generates a report, only the API Key prefix and suffix are displayed — for example: sk-****abcd. Full API Keys are never shown.

5. Data Use Limitations

AI API Doctor commits to the following:

  • Will not upload API Keys to AI API Doctor servers
  • Will not sell user data
  • Will not use user data for advertising or profiling
  • Will not use user data for credit scoring or lending decisions

6. Chrome Web Store Limited Use Statement

The use of information received from Chrome APIs will adhere to the Chrome Web Store User Data Policy, including the Limited Use requirements.

7. Safety Boundaries

AI API Doctor only shows reproducible signals from the diagnostic test. It does not prove intentional overbilling by the provider. Reports are best suited for troubleshooting, customer support communication, and owner repair — not as financial proof or legal grounds.
