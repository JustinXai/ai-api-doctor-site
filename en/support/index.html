Include These Details When Submitting an Issue

The following information helps us locate the issue quickly. Only provide necessary details — do not upload your full API Key.

  • AI API Doctor version number (visible in extension settings)
  • Browser version (e.g. Chrome 126)
  • Operating system (e.g. macOS 14, Windows 11)
  • Base URL domain (e.g. api.example.com — do not provide the full API Key)
  • Model ID (e.g. gpt-4o, claude-3-5-sonnet)
  • HTTP status (e.g. 503, 200)
  • Screenshot of the report or copied report text
  • Whether you were logged in to a New API / One API dashboard
  • Whether raw quota was readable
Do not submit your full API Key, account password, or sensitive balance screenshots publicly. Reports automatically desensitize API Keys (e.g. sk-****abcd). Please confirm the report text does not contain the full Key before submitting.

Scope of Support

AI API Doctor can help investigate:
  • Whether failed requests deduct raw quota
  • Whether empty replies are charged
  • Whether pre-charges are refunded within 10 seconds
  • Whether Base URL / API Key / Model ID are functional
  • How to send reports to the owner or support team
AI API Doctor cannot guarantee:
  • Proof of intentional overbilling by the provider
  • 100% coverage of all relay station templates
  • Replacement for a legal audit report
  • Reading balances on non-New API / non-One API stations
