Why Do You Need This Tool?

Many AI API relay providers use a pre-charge mechanism. When a request fails, returns empty, encounters a 503 from upstream, times out, or hits cache, it becomes hard for users to tell if the charge was fair. AI API Doctor helps by producing reproducible signals that users and providers can discuss based on evidence.

Five Diagnostic Dimensions

Billing Integrity
Detect whether failed requests and empty replies are ultimately charged. Wait 10s to confirm pre-charges are refunded.
Model Connectivity
Verify Base URL, API Key, model permissions, and HTTP status.
Usage Integrity
Check whether the response returns prompt_tokens, completion_tokens, and total_tokens.
Cache Hit
Send two identical requests and check whether cached_tokens / cache_read_input_tokens are returned.
Price Audit
Calculate expected cost from usage and prices, compare with actual deduction to find anomalies.

Why Did We Build AI API Doctor?

When I was building an AI API service, I discovered that many relay providers use a pre-charge mechanism: quota is deducted when a request starts, and settled based on actual usage after the request ends.

This mechanism itself is fine. But if failed requests, empty replies, upstream 503 errors, invalid models, or cache hits are not handled well, users may find that "they got no valid response but their quota changed".

AI API Doctor's goal is not to convict any site — it's to turn billing, usage, cache, and model performance into a reproducible diagnostic report that users and providers can discuss based on evidence.

AI API Doctor does not recommend or rank any relay providers. The report only shows reproducible signals from this test, and does not prove intentional overcharging by the provider.
