检测原理：API Doctor 怎么照出中转站有没有坑

直接发请求，看结果，照出有没有坑。

它怎么测？

直接向你填写的中转站地址发请求，然后看返回结果有没有异常。

它看哪些东西？

  • 有没有内容
  • 有没有账单明细
  • 用量有没有明显偏高
  • 流式请求有没有丢账
  • 缓存有没有透传
  • 模型表现有没有缩水

它不看哪些东西？

  • 不看宣传页
  • 不看站长承诺
  • 不读取你的后台余额
  • 不保存你的 API Key
  • 不证明谁一定造假
AI API Doctor 只展示本次测试中的可复现信号，报告更适合用于排查、客服沟通和站长修复。

为什么不能测真实扣费？

网页不能读取中转站控制台的原始额度。真实扣费后续由 Chrome 插件在控制台页面读取请求前后的额度变化。

如有疑问，建议记录多次测试报告，对比结果一致性，并联系站长或客服核对后台数据。
